# Known Risks

This file tracks stable risks that should influence planning.

## Product Scope Risks

- AI Brain Pro could be mistaken for a product-facing AI feature. It is currently SDLC-only.
- AI chat is explicitly out of MVP scope as the main user interface.
- Adding ordering, payments, delivery, authentication, or scraping would exceed current MVP boundaries.

## Architecture Risks

- Native production build readiness is not yet proven because there are no native `ios/` or `android/` folders.
- `dist/` is generated by web export and can create noisy diffs if committed unintentionally.
- Documentation can become stale if AI Brain copies full source docs instead of linking and summarizing.

## Data Risks

- MVP recommendation data is controlled mock/seed data, not production-verified menu coverage.
- User-submitted observations are future untrusted input and will need validation, rate limiting, and abuse controls.
- Production scraping is restricted by policy.

## Security Risks

- `.env.local` exists locally and must never be read into memory or copied into docs.
- API keys, tokens, credentials, authorization headers, and private user data must not be logged or persisted.

## Validation Risks

- Docs-only changes do not prove app health.
- App code changes require validation evidence, not just inspection.
- Mobile release readiness needs a future Expo/EAS or native build plan.

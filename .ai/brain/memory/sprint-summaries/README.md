# Sprint Summaries

Store sprint or phase summaries here when work completes.

Use `../../templates/sprint-summary-template.md` for new summaries.

Naming convention:

```text
YYYY-MM-DD-phase-or-sprint-name.md
```

Each summary should include goals, completed work, validation evidence, decisions, risks, follow-ups, and memory updates.

The optional helper can create a starter summary:

```bash
npm run brain:memory:update -- --type=sprint --title="Sprint or phase name" --summary="One sentence closeout."
```

Review and complete the generated summary before treating it as final memory.
